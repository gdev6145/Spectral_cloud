Security Policy

Last updated: April 2026

⚠ If you have discovered a security vulnerability, please report it privately — do not open a public issue. See the Reporting a Vulnerability section below.

Reporting a Vulnerability

Please report security vulnerabilities using one of the following channels:

  • GitHub Security Advisories (preferred) – Open a private advisory directly in the repository's Security tab.
  • Email – Contact the maintainers at security@spectral-cloud.local.

When reporting, please include:

  • A clear description of the vulnerability and its potential impact.
  • Steps to reproduce or a proof-of-concept (where safe to share).
  • The version or commit SHA where you observed the issue.
  • Any suggested remediation, if known.

We aim to acknowledge receipt within 2 business days and to provide a remediation timeline within 7 business days.

Supported Versions

Security patches are provided for the latest released version only. Users on older versions are encouraged to upgrade promptly.

Built-in Security Controls

Spectral Cloud includes the following security features:

  • API key authentication – All data-plane endpoints require a valid API key or tenant key. Configure via API_KEY, TENANT_KEYS, or TENANT_WRITE_KEYS.
  • Admin CIDR restriction – Admin endpoints can be restricted to trusted IP ranges using ADMIN_CIDR.
  • TLS support – Enable HTTPS via TLS_CERT / TLS_KEY. Mesh traffic supports mutual TLS via MESH_TLS_* environment variables.
  • Rate limiting – Both per-IP (RATE_RPS, RATE_BURST) and per-tenant (TENANT_RATE_RPS, TENANT_RATE_BURST) rate limits are configurable.
  • Circuit breaking – Automatic circuit breaking prevents unhealthy agents from receiving new work.
  • Backup encryption – Backup archives can be encrypted with AES-256-GCM using BACKUP_ENCRYPTION_KEY.
  • Audit logging – Authenticated write operations are recorded in the audit log.
  • Request size limits – Configurable maximum request body size via MAX_BODY_BYTES.

Dependency Security

The project uses govulncheck ./... as part of CI to identify known vulnerabilities in Go dependencies. To run this check locally:

  • Install: go install golang.org/x/vuln/cmd/govulncheck@latest
  • Run: govulncheck ./... in the repository root.

Some vulnerabilities may require upgrading the Go toolchain or gRPC libraries. Schedule upgrades promptly when security advisories are published for your dependency versions.

Responsible Disclosure

We ask that you follow responsible disclosure practices: give us reasonable time to investigate and patch before publicly disclosing the issue. We will credit reporters in the release notes (with your permission).
