Privacy Policy

Last updated: April 2026

Spectral Cloud ("the Service") is committed to protecting your privacy. This Privacy Policy explains what information is collected, how it is used, and how it is protected when you use the Service.

1. Information We Collect

The Service collects the following categories of information:

  • Account credentials – API keys and tenant identifiers used to authenticate requests. These are hashed or stored securely and are never logged in plaintext.
  • Operational data – Blockchain blocks, routing entries, agent registrations, job payloads, and other data you explicitly submit through the API.
  • Request metadata – HTTP request logs including timestamps, IP addresses, HTTP methods, paths, and response codes, used for auditing and debugging.
  • Health and metrics data – Prometheus-compatible metrics about service health, throughput, and latency. These contain no personally identifiable information.

2. How We Use Your Information

Information collected by the Service is used solely to:

  • Authenticate and authorize API requests.
  • Fulfill the operations you request (storing blocks, routing data, managing agents and jobs).
  • Monitor service health and detect anomalies or security incidents.
  • Produce operational audit logs for compliance and troubleshooting purposes.

The Service does not sell, rent, or share your data with third parties for advertising or marketing purposes.

3. Data Storage and Retention

All persistent data is stored in an embedded BoltDB database on the server where the Service is deployed. You, as the operator, control where this database is stored. Data is retained until explicitly deleted through the API, a restore operation, or manual removal of the data directory.

Access logs, if enabled via ACCESS_LOG=true, are written to standard output or a configured log file and are subject to your infrastructure's log retention policies.

4. Data Security

The Service implements the following security controls:

  • API key authentication for all sensitive endpoints.
  • Optional TLS encryption for all HTTP and mesh traffic.
  • IP-based and tenant-based rate limiting to prevent abuse.
  • Audit logging of authenticated write operations.
  • Optional backup encryption via AES-256-GCM when BACKUP_ENCRYPTION_KEY is configured.

Despite these measures, no system is perfectly secure. You are responsible for deploying the Service in a secure environment and maintaining the confidentiality of your credentials.

5. Your Rights

As a self-hosted service, you have full control over your data:

  • You can delete any data at any time using the API or by removing the data directory.
  • You can export your data using the backup functionality provided by spectralctl backup.
  • You can restrict access to your deployment using the built-in authentication and network controls.

6. Cookies and Tracking

The Spectral Cloud dashboard does not use cookies or any third-party tracking technologies. Theme preferences are stored in your browser's localStorage and never transmitted to the server.

7. Third-Party Services

The Service does not make outbound connections to third-party analytics, telemetry, or data-collection services. Any integrations (such as webhook endpoints you configure) are entirely under your control and subject to the privacy policies of those external systems.

8. Changes to This Policy

This Privacy Policy may be updated from time to time. The "Last updated" date at the top of this page reflects when changes were last made. Continued use of the Service after changes are posted constitutes acceptance of the revised policy.

9. Contact

For privacy-related questions or requests, please open an issue on the project repository or contact the maintainers via the channels listed in our Security page.
